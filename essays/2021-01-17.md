Software Engineering is a discipline that requires intense understanding of efficient programming techniques and syntax. Therefore, in order to be a successful software engineer in the future, it is important to realize that “memorizing” code is not the best solution to solving a coding problem. It is essential to know what you are typing and how the system reads and processes that code. That way, if you were to encounter a similar problem in another program, you will be able to understand the process that allowed you to solve the problem, ultimately saving time. There are many resources available to a programmer to simply search up the answer, but this will have a negative impact on them in the long run because it was not their own work. While it can be useful to seek assistance from others, it is important to get help on the process rather than the answer. This is what Eric Raymond talks about in his essay, “How to Ask Questions the Smart Way”. StackOverflow is an online resource available for everyone to ask questions regarding their code. However, there are “smart” questions and “not smart” questions that could affect the type of responses one may receive. 

## Smart

<img class="ui small left floated image" src="../images/essay3-thumbsup.png">

One of the most important points that Raymond talks about is to ask questions by describing the goal first, and then the steps you are having an issue on, rather than simply asking why your code does not work. This is important because it ensures that you will get a specific answer to your issue. You will receive assistance on the part that you are stuck on rather than reading a full explanation of code that you possibly already knew how to do. A “smart” question asked on StackOverflow was “why is processing a sorted array faster than processing an unsorted array?”. This user was not asking how to solve a coding problem, but rather encountering a runtime issue after completing the code, and is asking the question hoping to understand why this is the case. They gave detailed explanations that included the seconds it took to run the code, an attempt to find a solution by testing the code in C++ and Java to see if it had to do with the language, another idea which involved data cache but ruled it out, and also uploading their code in C++ and Java as reference. The user was smart to try to figure out a solution on their own first, and then posted their question when they could not figure it out. With smart questions comes helpful responses; the top response was able to effectively answer the question with a simple analogy that connected to the code. 

You can view the StackOverflow post [here](https://stackoverflow.com/questions/11227809/why-is-processing-a-sorted-array-faster-than-processing-an-unsorted-array)

## Not Smart

<img class="ui small left floated image" src="../images/essay3-thumbsdown.png">

A “not smart” question involves seeking for the final solution without asking for guidance towards the final solution. These are usually very brief questions with no indication of any previous attempts the user tried before posting. An example of a “not smart” question asked on StackOverflow would be “How to count sum of all indexes of multiple array and store it to specific index”. This post only consisted of one question and a screenshot of two arrays. There were no other explanations, which made the problem more difficult to understand. For example, it is difficult to truly know what the user meant by a “specific index” because they did not say what the index was for. 

You can view the StackOverflow post [here](https://stackoverflow.com/questions/65787378/getting-sum-of-all-fields-of-multiple-arrays)

## Takeaways

After reading Eric Raymond’s essay on asking smart questions, I think that it is very insightful because it could be another way of learning. It is important to always try to solve problems on your own first before seeking help from outside sources. If it does come to the point where I need to ask for help, I will know how to ask the questions. I know not to simply ask “Help!” but rather asking to point me in the right direction. I will try my best to make sure the people helping me do not explicitly tell me the solution because that will not help me to learn on my own.
