Back when I was a kid browsing the internet, I have always wanted to create a website of my own merely because I thought it would make me look cool. I remember attempting to create websites using free tools such as Google Sites, Wix, and Weebly, but it would always leave their domain in the URL which, in my opinion, made the website less cool because it was essentially a watermark. Back then, the only thing that made me look cool was going into Chrome, right clicking, clicking inspect element, changing the text of a webpage, and showing it off to my friends. Now that I am thinking about it, this was not very cool at all.

## Back to the 1990s

When the time came to learn about HTML and CSS, my brain jumped right into learning all about it because I knew this was the time I can finally personalize my own website. I guess it was also to make up for my struggles attempting to make a website long before I knew anything about basic programming. I was ready to make a beautiful website with interactive user interfaces. But I was arrogant. This was much more difficult than I thought. Trying to create a website purely out of raw HTML and CSS was a lot less appealing than I thought. It looked like a discontinued website from the 1990s. There was poor spacing, resizing, and overall it looks plain compared to the modern websites today. I am sure that if I get deep into understanding HTML and CSS, I should be able to create a much better looking website. While the use of classes makes grouping elements easier, the difficulties of customizing these elements from scratch far outweighs the power of using classes. 

Is this similar to how it feels like to code in Assembly after knowing how to code in more modern languages? That was a rhetorical question; don’t answer it.

## Webpage Beautification

I am kind of glad I did not get too deep into creating websites purely from raw HTML and CSS. When I began learning about Semantic UI, I thought that this was a more complicated extension to HTML and CSS, but it was actually the complete opposite. The power of UI Frameworks makes customizations much easier due to the premade classes that essentially do the hard work for me. Rather than having tens and hundreds of lines of raw HTML to display one element such as a menu bar, Semantic UI is able to display the same element in a few lines of code and simply typing the word "menu".

## UI Frameworks are Great

Website modifications are made much easier with the help of UI Frameworks. As a software engineer, it is important to make your code easy to understand so you or another developer can revisit the source code and make necessary changes in the future. Rather than changing a lot of the properties in raw HTML and CSS, UI Frameworks simplifies it so you would only need to change a few words in the “class” property. I find it much more fun to create a website out of Semantic UI, probably because it is a lot easier to code and I somewhat know what I am doing.
